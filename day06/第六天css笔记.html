第六天css笔记

想要让块级元素在一行显示，转换为行内块出现的问题：

1、如果代码中出现了回车换行，页面上元素之间就会出现间隙

解决的办法：

  • 1、消除代码中回车符换行符，把代码变到一行显示

  • 2、给它的父级的font-size:0;然后再具体根据设计稿给里面的子元素设置字体大小。

第一种：

<body>

    <div class="box1">1</div><div class="box2">2</div>

</body>

第二种：

<style>

        body{

            font-size:0

        }

        *{

            margin:0;padding:0;

        }

        div{

            width:100px;

            height:100px;

            background:green;

            display: inline-block;

            font-size:14px;

        }

    </style>



<body>

    <div class="box1">1</div>

    <div class="box2">2</div>

</body>

如果有两个div，转换成行内块，里面有一个有文字，有一个没有，行内块元素默认是基线对齐，两个div就不能在顶部对齐

解决办法：改变vertical-align 的值，改成top或者middle

<style>

        body{

            font-size:0

        }

        *{

            margin:0;padding:0;

        }

        div{

            width:100px;

            height:100px;

            background:green;

            display: inline-block;

            font-size:14px;

            vertical-align:top;

        }

    </style>



<body>

    <div class="box1"></div>

    <div class="box2">2</div>

</body>

浮动

浮动的用法：想让谁浮动，我们就给谁添加float：XXX；

  • float：left 代表左浮动

  • float：right 代表右浮动

  • float：none

浮动的特点：

  • 浮动的元素，是脱离文档流的。

  • 浮动元素的参照物是它的父级

  • 一旦给元素设置浮动，浮动的元素如果不设置宽度，它不再像原来一样继承父级的宽度，宽度是由自身的内容决定的。

  • 容易造成“盒子塌陷”

  • 浮动最初的本质是做图文混排，就是一个图片，一段文字，一旦给图片设置浮动，文字就会环绕着图片排列。

    • 如何解决浮动造成的盒子塌陷问题？

    • 给浮动元素的父级添加高度，height；

    • 给浮动元素的父级添加overflow:hidden;

    • 给浮动元素的父级末尾添加一个盒子，然后给这个盒子设置clear：both；清除左右浮动

    • 用clearfix;把下面的.clearfix 代码每次写项目的时候，都给放到样式表里，只要有元素浮动了，咱们立马给它的父级添加类名：clearfix，这样浮动元素的父级就不会受子元素浮动的影响了。（这个方法简单，好用，项目中现在都用这个，它的本质和第三个一样的，只是第三个是手动在父级末尾添加盒子，给盒子设置clear：both，这种方法，不用手动添加盒子，是用伪类的方法添加的，这样用起来很方便）

           .clearfix:after{

                display: block;

                content: "";

                visibility: hidden;

                font-size:0;

                height:0;

                clear:both;

            }

            .clearfix{

                *zoom:1;

            }
