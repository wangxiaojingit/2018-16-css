第十四天css笔记

产品形态

  • pc端和移动端公用一个域名的，就是共用一套代码，比如：华为、猎豹、苹果

  • pc端和移动端分开的，pc端单独一套，移动单独一套。例如：淘宝、京东…

什么时候共用一套，什么时候单独分开？

当网页的结构、内容比较简单的时候，我们可以共用一套，如果网页的内容比较复杂，共用的时候，会写大量的代码，不好修改，不好维护，而且还减慢了加载速度，此时我们分两套写，例如京东、淘宝

pc端和移动端的区别

pc 端有鼠标相关的事件，比如点击，移动端没有，但是移动端有滑屏、触摸当相关事件
pc端需要兼容ie等浏览器，移动不需要考虑这些问题，它的内核和谷歌一样，都是webkit，但是移动端我们需要考虑适配问题，还有少许的样式兼容。（手机型号不同的时候，样式会有细微的差别）

调文字间距

  • letter-spacing： 字符与字符之间的间距

  • word-spacing: 单词与单词之间的间距（中间有空格隔开，代表的是一个单词）

   -moz-border-radius：50%；//火狐

   -o-border-radius：50%；//欧朋

box-shadow,阴影

.box{

            width:200px;

            height:200px;

            background:darkorange;

            border-radius:100px;

            /*

             第一个值：偏离x轴的距离

             第二个值：偏离Y轴的距离

             第三个值：阴影模糊度

             第四个值：阴影大小

             第五个值：阴影的颜色

             第六个值：inset:内阴影，不写的时候是outset 外阴影

            */

            box-shadow:0px 0px 100px 0px rgba(0,255,139,.5) inset;

        }

视口

在写移动端页面的时候，如果不加视口，html默认的宽度是980px,如果模拟iphone5的时候，手机屏幕的宽度是375px;把一个980的页面放到375的容器中，要想放的下，肯定要缩小，所以看不清楚，所以在做移动端的时候我们必须加上视口。

<meta name="viewport"

          content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">

  • name=”viewport” 视口

  • width=device-width: 让页面（html）的宽度等于设备的宽度

  • user-scalable=no：禁止缩放

  • initial-scale=1.0；初始缩放值为1.0

  • maximum-scale=1.0；最大的缩放值也为1.0

  • minimum-scale=1.0；最小的缩放值也为1.0

媒体查询

  • media (媒体媒介)

    • screen：屏幕（主要是screen）

    • all：所有媒体

    • print：打印机

    • …. 剩下的都是不常用的

    • 连接符:and,前后一定要空格，不然不起作用

       //max-width: 最大宽度，就是小于等于767px；

        @media screen and (max-width:767px) {

            ul li{

                width:100%;

            }

        }



    //min-width:最小宽度，大于等于

     @media screen and (min-width:1200px){

            ul>li{

                width:25%;

            }

        }
